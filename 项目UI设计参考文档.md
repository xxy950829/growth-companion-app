# 童伴星球 App - UI设计参考文档

> 本文档详细描述现有UI的所有元素、页面结构、交互逻辑和视觉规范，供设计师进行UI重设计参考。

---

## 一、项目概述

**产品名称**：童伴星球  
**目标用户**：年轻父母（0-6岁宝宝家长）  
**核心价值**：陪伴式宝宝成长记录工具，用游戏化方式培养育儿习惯  

### 三大核心模块

| 模块 | 功能定位 | 核心概念 |
|------|----------|----------|
| 成长档案 | 记录宝宝里程碑 | 时间轴 + 分类记录 |
| 成长星球 | 习惯任务养成 | 升级体系 + 游戏化反馈 |
| 心情天气瓶 | 情绪记录 | 天气隐喻 + 可视化 |

## 三、页面流程与路由结构

### 3.1 路由架构

```
App启动
  ↓
[Loading] 加载状态
  ↓
├─ 未登录 → 登录页 ↔ 注册页
│
└─ 已登录
   ├─ 无宝宝档案 → 引导页（创建宝宝档案）
   │
   └─ 有宝宝档案 → 底部Tab导航
      ├─ 📚 成长档案 (archive)
      ├─ 🪐 成长星球 (planet)
      ├─ 🌤️ 心情天气 (weather)
      └─ 👤 个人中心 (profile)
```

### 3.2 页面切换逻辑

- **Header区域**：点击左上角宝宝头像区域可快速切换宝宝（多宝宝场景）
- **浮动按钮**：成长档案、心情天气页面右下角有圆形"+"浮动按钮，用于新增记录
- **底部弹窗**：所有表单（新增里程碑、新增任务、记录心情）均以底部滑出Modal呈现

## 四、基础UI组件库

### 4.1 Button 按钮

**文件**：[Button.tsx](file:///workspace/components/ui/Button.tsx)

**变体（Variant）**：
- `primary`：主按钮，背景色 `#FF8C5A`，白色文字
- `secondary`：次要按钮，背景色 `#7BC9A6`，白色文字
- `ghost`：幽灵按钮，透明背景，橙色边框+文字
- `danger`：危险按钮，背景色 `#EF4444`，白色文字

**尺寸（Size）**：
- `sm`：padding 8/14，字号13px
- `md`：padding 12/20，字号15px（默认）
- `lg`：padding 16/28，字号17px

**状态**：默认、loading（显示ActivityIndicator）、disabled（透明度0.5）

---

### 4.2 Card 卡片

**文件**：[Card.tsx](file:///workspace/components/ui/Card.tsx)

- 白色背景 `#FFFFFF`
- 圆角 20px
- 内边距 16px
- 柔和阴影（y偏移4px，模糊12px，透明度0.08）
- Android elevation: 3

### 4.3 Input 输入框

**文件**：[Input.tsx](file:///workspace/components/ui/Input.tsx)

**结构**：
```
Label（可选）
└─ 输入框容器（bg2背景 + 圆角12px + 边框）
   ├─ 左侧图标（emoji，可选）
   └─ TextInput
Error提示（可选，红色）
```

**特性**：
- 支持 label、leftIcon、error 状态
- 错误状态边框变红 `#EF4444`
- placeholder文字颜色为 muted

---

### 4.4 Modal 底部弹窗

**文件**：[Modal.tsx](file:///workspace/components/ui/Modal.tsx)

**呈现方式**：从底部滑出（slide animation）
- 遮罩层：黑色半透明 `rgba(0,0,0,0.4)`
- 弹窗背景：白色，上圆角24px
- 最大高度：屏幕85%-90%
- 顶部标题栏：标题 + 关闭按钮(✕)
- 支持 footer 区域（放置取消/保存按钮）
- 点击遮罩可关闭

---

### 4.5 Header 顶部导航栏

**文件**：[Header.tsx](file:///workspace/components/ui/Header.tsx)

**结构**：
```
[ 左侧区域 ]                    [ 右侧区域 ]
宝宝头像 + 宝宝姓名              页面标题
        + 宝宝年龄 · 共N个宝宝          副标题
（点击可切换宝宝）
```

**特性**：
- 固定在页面顶部，paddingTop: 48px（适配状态栏）
- 背景色与页面一致（无分割线，沉浸式）
- 多宝宝时点击左侧可循环切换
- 右侧支持自定义right元素（如设置按钮）

### 4.7 FeedbackHost 全局反馈

**文件**：[FeedbackHost.tsx](file:///workspace/components/ui/FeedbackHost.tsx)

**组件一：Toast 顶部提示**
- 位置：顶部60px处，左右边距24px
- 类型：success(绿✓)、error(红✕)、info(橙ℹ)
- 动画：顶部滑入+弹性效果
- 结构：左侧彩色竖条 + 图标 + 消息文字
- 自动消失

**组件二：ConfirmDialog 确认对话框**
- 居中弹窗，宽度80%（最大340px）
- 圆角20px，带缩放动画
- 支持多个按钮（默认/主要/危险三种样式）
- 点击遮罩可关闭

## 七、交互模式总结

### 7.1 通用交互

| 交互场景 | 交互方式 |
|----------|----------|
| 新增记录 | 右下角FAB浮动按钮 → 底部弹窗表单 |
| 删除操作 | 卡片上的"删除"文字 → 确认对话框 → 执行删除 |
| 切换宝宝 | Header点击头像区域 → 直接循环切换 |
| 选择照片 | 点击"+添加" → ActionSheet（拍照/相册/取消）→ 上传 |
| 表单提交 | 底部"保存"按钮 → loading状态 → 关闭弹窗 + Toast提示 |
| 筛选切换 | 横向Chip/Tag，点击即切换 |

### 7.2 反馈机制

- **操作成功**：顶部Toast滑入（绿色✓），2-3秒后自动消失
- **操作失败**：顶部Toast滑入（红色✕），显示错误信息
- **危险操作**：居中确认对话框，红色危险按钮
- **空状态**：大emoji + 友好提示文字引导用户创建

### 7.3 列表展示模式

| 模块 | 列表形式 |
|------|----------|
| 成长档案 | 时间轴（按日期分组，左侧连接线） |
| 任务列表 | 卡片列表（勾选框+图标+信息） |
| 心情历史 | 卡片列表（左侧彩色竖条） |
| 宝宝列表 | 卡片列表（头像+信息+编辑按钮） |

## 八、现有UI问题（设计师可参考优化方向）

1. **视觉层次单一**：大量使用圆角卡片，层次不够分明
2. **emoji为主**：图标几乎全用emoji，缺少品牌化设计
3. **色彩偏平**：虽然是暖色调，但缺少渐变、阴影层次等细节
4. **数据可视化简单**：星球、天气瓶等核心创意组件用纯View+emoji实现，视觉表现力有限
5. **交互反馈弱**：缺少微动效、弹性动画、转场动画
6. **表单较朴素**：日期选择、时间选择等控件是自定义简单实现，体验可优化
7. **空状态设计简陋**：仅emoji+文字，缺少插画引导
8. **缺少引导页/欢迎页**：登录后直接到档案创建，缺少品牌展示
9. **Tab栏emoji图标**：应替换为定制图标
10. **天气瓶可增强**：作为核心创意点，可以加入液体波动、粒子效果等动画
